1. Resumo do Projeto

Este documento apresenta a solução técnica desenvolvida para o Tech Challenge, que consiste em um sistema de autogerenciamento de pedidos para uma rede de restaurantes. A solução foi projetada com base em uma arquitetura de microserviços, visando alta escalabilidade, resiliência e manutenibilidade.

O sistema permite que os clientes se cadastrem, façam pedidos, acompanhem o status e realizem o pagamento de forma totalmente digital. Do lado do restaurante, a plataforma oferece ferramentas para o gerenciamento de cardápios e o recebimento de pedidos.

2. Arquitetura da Solução

A arquitetura do sistema é fundamentada em princípios de microserviços, com cada serviço sendo responsável por uma funcionalidade de negócio específica. A comunicação entre os componentes é orquestrada por um API Gateway e um sistema de mensageria, garantindo desacoplamento e flexibilidade.

2.1. Ecossistema de Microserviços

  • Identity Service: Responsável pelo ciclo de vida dos usuários (cadastro, autenticação, autorização) e pela emissão de tokens JWT para proteger as APIs.
  • Restaurant Service: Gerencia todas as informações relacionadas aos restaurantes, como cardápios, mesas e horários de funcionamento.
  • Order Service: Orquestra o fluxo de criação e acompanhamento de pedidos, desde a seleção dos itens até a confirmação final.
  • Payment Service: Processa os pagamentos de forma assíncrona, integrando-se com gateways de pagamento externos e tratando cenários de falha com uma lógica de retentativas.

2.2. Componentes de Infraestrutura

  • Kong (API Gateway): Atua como ponto de entrada único para todas as requisições, centralizando a autenticação, o roteamento para os serviços internos e a aplicação de políticas de segurança.
  • RabbitMQ (Message Broker): Facilita a comunicação assíncrona e confiável entre os microserviços, sendo crucial para fluxos como a confirmação de pedidos e o processamento de pagamentos.
  • PostgreSQL (Banco de Dados): Cada microserviço possui sua própria instância de banco de dados, garantindo isolamento de dados e autonomia para cada serviço.
  • Docker: Todos os serviços e componentes de infraestrutura são containerizados, e a orquestração do ambiente é gerenciada com o Docker Compose, garantindo portabilidade e consistência entre os ambientes.

4. Como Executar o Projeto

Para executar o ambiente completo em uma máquina local, siga os passos abaixo:

  1. Pré-requisitos:
    • Docker e Docker Compose instalados.
    • Arquivo .env configurado na raiz do projeto de infraestrutura (um arquivo .env.example é fornecido como modelo).
  2. Execução: 
    # Clone o repositório de infraestrutura
git clone https://github.com/KervinCandido/restaurant-system-infra.git
cd restaurant-system-infra

# Suba todo o ecossistema de serviços
docker-compose up -d

5. Conclusão

O projeto entregue representa uma solução robusta e moderna para o problema proposto, aplicando conceitos avançados de arquitetura de software, como microserviços, comunicação assíncrona e design orientado a domínio. A solução está pronta para ser expandida com novas funcionalidades e para ser implantada em um ambiente de produção em nuvem.
